Demo Application

The RAP project comes with two example applications demonstrating the features and capabilities of RAP.

The first application provides a workbench very closely aligned to the RCP workbench. The second demo shows what control widgets there are and that they behave very similar to the SWT widgets.

To access and run these demo applications, please open the Plug-ins view (e.g. through the Plug-in Development perspective). Select the plugin named org.eclipse.rap.demo which contains both demo applications, right-click on this plugin and select Import as > source project . This demo will then appear with all source files in your workspace. The demo also provides a ready-to-use launch configuration.

Please note that even though importing the demo project as source succeeds, the plug-in can only be run if You installed the SDK release. When using the Update-Manager to get RAP, the Rich Ajax Platform SDK component must also be checked as shown below.

To run the demo, select Run. In the upcoming Launch Configuration Dialog, select Rich AJAX Platform Demo from the section Equinox OSGi Framework.

Important note for Eclipse 3.3:
If you are using Eclipse 3.3. already, you need to start the org.eclipse.equinox.app plugin with your rap demo. To do so, switch to the Plug-ins tab of your Launch Configuration and select to include the org.eclipse.equinox.app plug-in.

You may want to configure the TCP-port your demo application will listen to. This is done opening the Arguments tab and passing -Dorg.osgi.service.http.port=10080 as an argument to the Java Virtual Machine (VM), where 10080 is the (arbitrarily chosen) port your application will listen to. This is particularly important if the default port 80 is already taken by some other application (e.g. your Web Server ...) or in an environment where you are not allowed to access privileged ports (e.g. as a non-admin user on some operating systems).

To actually start your demo, click the Run button. The correct startup should display in the Eclipse Console through a message saying INFO: Started SocketListener on 0.0.0.0:<port>. If an error shows up, please make sure all other instances are stopped and then retry with another port.

Now that the rap demo application is running, you can access it using any browser. Please enter the URL http://localhost:<port>/rap.

The rap demo application defines two entry points. The default is the workbench demo. You can now select the controls-demo by explicitly passing a parameter in the url like this:
http://localhost:<port>/rap?w4t_startup=controls

The controls demo will show you all the widgets that come with RWT and how they work just like SWT in a browser.
